# Commit Message Guidelines

## Common types

| **Type**     | **When to Use**                                            | **Example**                              |
| ------------ | ---------------------------------------------------------- | ---------------------------------------- |
| **feat**     | When adding a new feature or capability                    | `feat(auth): add JWT refresh token`      |
| **fix**      | When fixing a bug or regression                            | `fix(db): correct null value issue`      |
| **docs**     | For documentation changes only                             | `docs: update setup instructions`        |
| **style**    | Code formatting, style, or linting (no code logic change)  | `style: run Biome format`                |
| **refactor** | Code cleanup or restructuring without changing behavior    | `refactor(api): simplify error handling` |
| **perf**     | Performance improvements                                   | `perf(api): cache heavy queries`         |
| **test**     | Adding or updating tests                                   | `test(auth): add login validation tests` |
| **build**    | Changing build tools, dependencies, or scripts             | `build: add pnpm workspace support`      |
| **ci**       | Updating CI/CD workflows or pipelines                      | `ci: add GitHub Actions test job`        |
| **chore**    | Project setup, maintenance, or development process updates | `chore: setup Lefthook and Commitlint`   |
| **revert**   | Reverting a previous commit                                | `revert: remove broken Docker setup`     |
